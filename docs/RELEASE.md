# Skill Studio Pro Release Guide

Repository: https://github.com/Hao2080/skill-studio-pro

## Release model

- `main` is the default branch.
- `.github/workflows/ci.yml` runs frontend/Rust gates, native Secret Store contracts, Tauri packaging, installed launch smoke, SBOM/license policy, dependency audits, repository hygiene, and full-history secret scanning.
- `.github/workflows/release.yml` can be run manually to validate release packages without publishing. Pushing the exact tag `v0.1.0-beta.1` repeats the three-platform gates, verifies downloaded artifact hashes, and creates a prerelease.
- Release assets are generated by GitHub-hosted runners and are never committed as installers.

## Version and local gate

The following values must be exactly `0.1.0-beta.1`:

- `package.json` and `package-lock.json`
- `src-tauri/Cargo.toml` and `src-tauri/Cargo.lock`
- `src-tauri/tauri.conf.json`
- Release tag `v0.1.0-beta.1`

```bash
npm ci
npm run check
npm run supply-chain:check
npm audit --audit-level=high
git diff --check
```

`npm run supply-chain:check` verifies the lockfiles, three reproducible CycloneDX 1.6 SBOMs, the dependency/license policy, repository links, version alignment, upstream/origin isolation, and disabled updater state.

## Beta signing boundary

No paid or user-owned signing identity is required for this beta:

- Windows NSIS is not Authenticode-signed.
- macOS DMG is not Developer ID-signed or notarized. Tauri/platform tooling may apply ad-hoc metadata; that is not publisher identity signing.
- Linux AppImage/deb are not distribution-signed.
- `createUpdaterArtifacts` is `false`; no updater plugin, public key, endpoint, `latest.json`, or updater signature is produced.

Never reuse upstream signing keys or release endpoints.

## Publish

Only tag a clean, committed `main` revision after the main CI run is green:

```bash
git tag -a v0.1.0-beta.1 -m "Skill Studio Pro v0.1.0-beta.1"
git push origin v0.1.0-beta.1
```

Do not move or overwrite a published tag. If a public beta has a defect, fix it on `main` and publish the next prerelease version.

## Required release assets

- Windows NSIS installer
- macOS DMG
- Linux deb and AppImage
- Per-platform artifact manifests and checksum files
- Per-platform quality-gate, installed-startup smoke, and native Secret Store evidence
- `SHA256SUMS.txt` and `release-manifest.json`, verified again after Actions artifact download
- Frontend, Rust, and combined CycloneDX JSON SBOMs
- Machine-readable and human-readable third-party dependency/asset license inventories
- `LICENSE.txt` and `NOTICE.txt`

After publication, read the public repository, Actions run, tag, Release, and asset list again. Download the assets, verify `SHA256SUMS.txt`, and confirm the tag targets the intended `main` commit.

## Troubleshooting

| Failure | Required response |
|---|---|
| Rust/TypeScript gate fails on one OS | Read that job log, add a regression test where applicable, fix, push, and rerun the full matrix |
| Native Secret Store fails | Check the isolated Credential Manager/Keychain/D-Bus Secret Service harness; do not replace it with a plaintext or mock persistence claim |
| Desktop process exits during smoke | Inspect the uploaded smoke log and platform dependencies; proving that a package file exists is not sufficient |
| Hash verification fails | Stop publication, compare the platform manifest with downloaded bytes, and rebuild from a clean tagged commit |
| SBOM/license policy fails | Resolve stale generated files, unknown declarations, or incompatible licenses; never invent a license |
| GitHub token cannot create a Release | Keep all local and Actions evidence, report the exact permission blocker, and do not claim publication |
