# Skill Studio Pro 产品需求文档（PRD）

文档状态：Draft 0.2（V1 产品闭环修订）
日期：2026-07-16
依赖文档：[SPEC.md](./SPEC.md)

## 1. 产品摘要

Skill Studio Pro 是一个跨平台桌面应用，用来发现、理解和管理电脑上的 AI Agent Skill。它把分散在 Codex、Claude Code、Cursor、Windsurf、Gemini CLI、插件缓存和项目目录中的 Skill 汇总到一个界面中，让用户知道每个 Skill 是什么、怎么用、在哪里、从哪里来、是否存在重复或风险，并能通过中央库把同一个 Skill 安装到多个 Agent。

产品以开源 Skill Studio 为基础，优先复用其本地工作区、快照、平台中心、导入、市场和同步能力。Skill Studio Pro 的差异化重点是“全机盘点、来源可信度、AI 透明、多 Agent 中央管理和安全回收站”。

## 2. 背景与问题

AI Agent 的 Skill 通常以目录和 Markdown 文件存在。随着用户同时使用多个 Agent，Skill 会散落在不同隐藏目录、插件缓存和项目中，形成以下问题：

1. 用户记不清安装了哪些 Skill。
2. 同一个 Skill 在多个 Agent 中有多份拷贝，内容逐渐不一致。
3. 只看名称无法判断用途，打开每个 `SKILL.md` 又很费时间。
4. Skill 被复制或手工安装后，来源、仓库和版本经常丢失。
5. 用户不敢删除旧 Skill，因为不知道是否仍被某个 Agent 使用。
6. 外部 Skill 可能包含脚本和敏感操作，但普通目录浏览无法快速提示风险。
7. 现有工具多偏向“安装”，缺少面向本机资产的完整盘点和可恢复管理。

## 3. 目标用户

### 3.1 多 Agent 个人用户

同时使用 Codex、Claude Code、Cursor 等工具，希望统一管理 Skill，不再手工复制文件。

### 3.2 Skill 收藏与重度使用者

安装了几十到几百个 Skill，需要搜索、分类、去重、来源追踪和版本管理。

### 3.3 Skill 作者

编写和维护自己的 Skill，需要文件编辑、快照、差异对比和跨 Agent 发布。

### 3.4 开源维护者和审计者

需要知道 Skill 来自哪里、包含哪些文件、是否有脚本，以及版本如何变化。

## 4. 用户核心诉求

用户希望打开软件后直接得到以下答案：

- 我的电脑上到底有多少 Skill？
- 每个 Skill 是干什么的，怎么用？
- 它安装在哪些 Agent 中？
- 它从哪里来，判断可靠吗？
- 有没有同名、重复或内容冲突？
- 哪些 Skill 包含脚本或外部依赖？
- 能不能只管理一份，然后同步到多个 Agent？
- 修改错了或删错了，能不能恢复？
- AI 生成的说明是哪个模型写的，是否已经过期？

## 5. 产品原则

1. **先看清，再操作**：盘点和解释必须先于安装、覆盖和删除。
2. **现有文件不乱动**：首次扫描只登记，不自动迁移用户文件。
3. **一个主版本，多处使用**：纳入中央库后，由主副本向多个 Agent 发布。
4. **删除可反悔**：默认进入回收站，不直接永久删除。
5. **事实和推断分开**：来源证据、推断和 AI 内容必须有清晰标记。
6. **AI 可见、可关、可替换**：显示模型职责，未配置 API 时核心功能照常可用。
7. **复用成熟能力**：上游已有功能优先改造，不重复开发。

## 6. 产品目标

### 6.1 第一代目标

- 完成本机多 Agent Skill 盘点。
- 建立中央 Skill 库和一对多映射。
- 支持安全安装、编辑、同步、删除和恢复。
- 提供来源证据、可信度和用户纠正。
- 提供 MiniMax 信息采集与 GPT-5.6 内容提炼。
- 建立模型配置、任务路由和结果溯源界面。
- 发布 Windows、macOS、Linux 开源版本。

### 6.2 不在第一代实现

- 账号系统
- 云同步和云备份
- 团队空间、成员、审批和合并请求
- 托管式在线 Skill 市场
- 自动执行 Skill 脚本
- 基于遥测的用户画像或远程分析

## 8. 页面需求

### 8.1 总览页

目的：打开应用后立即理解本机 Skill 状况。

显示：

- Skill 总数和实例总数
- 中央库数量
- 已检测 Agent 数量
- 来源已确认、推断、未知数量
- 重复和冲突数量
- 包含脚本或风险提示的数量
- 最近扫描状态
- MiniMax 和 OpenAI 连接状态
- 最近安装、编辑、同步和删除操作

主要操作：开始扫描、查看冲突、配置平台、配置模型。

### 8.2 本机 Skill 页

目的：展示从电脑各处发现的所有外部实例。

布局：左侧筛选器，中间列表或卡片，右侧可选快速详情。

列表字段：

- Skill 名称
- AI/本地简短说明
- 所在平台
- 来源和可信度
- 中央库状态
- 脚本/风险标记
- 重复/冲突状态
- 最近修改时间

主要操作：查看详情、纳入中央库、忽略、重新扫描、批量分类。

### 8.3 Skill 详情页

目的：在一个页面中看清 Skill 的内容、来源、分布和历史。

建议标签页：

- 概览
- `SKILL.md`
- 文件
- 来源
- 安装位置
- 版本与差异
- 操作记录

概览区必须显示：

- 一句话说明
- 使用建议
- 本地元数据
- AI 结果归属
- 来源可信度及证据
- 安装平台
- 风险提示
- 摘要是否过期

### 8.4 中央库页

目的：管理 Skill 的唯一主版本。

显示：

- 中央 Skill 列表
- 当前生效快照
- 已发布平台
- 平台漂移状态
- 未发布修改
- 最近同步结果

主要操作：创建、导入、编辑、创建快照、发布、从平台移除、移入回收站。

### 8.5 平台中心

目的：管理 Agent 检测、目录和同步方式。

每个平台显示：

- 平台名称和图标
- 是否检测到
- 是否启用
- Skill 目录
- 复制/符号链接模式
- 项目级支持状态
- 最近同步时间
- 当前受管 Skill 数量

首要平台：Codex、Claude Code、Cursor、Windsurf、Gemini CLI。

### 8.6 发现与安装页

目的：从本地、Git 和已有市场来源导入 Skill。

入口：

- 本地目录
- ZIP 文件
- GitHub/Git URL
- 市场浏览

安装前显示 Skill 数量、目标名称、来源、commit、脚本、文件数量、冲突和安装计划。默认先进入中央库，再由用户选择发布到哪些 Agent。

### 8.7 编辑工作区

目的：安全修改中央 Skill。

提供：

- 文件树
- 文本编辑器
- Markdown 预览
- 未保存状态
- 保存前恢复点
- 差异查看
- 外部编辑器入口
- 保存后发布选择

编辑外部扫描实例时，应用应先提示将其纳入中央库；不得默认直接修改第三方插件缓存或系统 Skill。

### 8.8 回收站页

目的：恢复误删 Skill 或执行明确的永久删除。

显示：名称、原路径、来源、删除时间、原映射和删除原因。主要操作为恢复、恢复到新名称和永久删除。

### 8.9 操作记录页

目的：回答“刚才发生了什么”。

支持按操作、Skill、平台、结果和时间筛选。失败记录应提供可重试入口或下一步建议。

### 8.10 设置页

分区：

- 通用：语言、主题、启动行为
- 扫描：扫描根、忽略规则、文件监听
- 中央库：数据目录、快照策略
- 平台：自定义平台和默认同步模式
- 模型与 API：MiniMax、OpenAI、模型职责和连接测试
- 隐私：自动 AI、发送前预览、敏感文件排除
- 开源与关于：版本、许可证、上游归属

## 9. 关键用户流程

### 9.1 首次启动

1. 用户打开 Skill Studio Pro。
2. 应用介绍本地优先、扫描范围和“首次扫描不会修改文件”。
3. 应用检测 Agent 目录。
4. 用户确认或调整扫描根。
5. 应用扫描并持续显示进度、当前目录和发现数量。
6. 完成后展示总览和需要处理的冲突。
7. AI 未配置时提示“可选配置”，但不阻挡使用。

成功标准：用户无需配置 API 即可看到本机 Skill 列表和原文。

### 9.2 查看一个 Skill

1. 用户从列表搜索 Skill。
2. 打开详情。
3. 先看到一句话说明、来源可信度、平台和风险。
4. 点击查看 `SKILL.md` 原文。
5. 若摘要不存在，用户可点击“生成说明”。
6. 生成后显示 MiniMax/GPT-5.6 的模型和职责信息。

### 9.3 将现有 Skill 纳入中央库

1. 用户在外部实例详情点击“纳入中央库”。
2. 应用展示来源目录、目标名称、冲突和计划。
3. 用户确认。
4. 应用复制文件、建立基线快照和来源关系。
5. 原实例保持不动。
6. 用户可选择把中央版本发布到其他 Agent。

### 9.4 从 GitHub 安装

1. 用户输入仓库 URL，可选分支、commit 和子目录。
2. 应用下载到暂存区并发现 Skill。
3. 显示安装预览、来源、脚本和冲突。
4. 用户选择导入项和目标 Agent。
5. 应用先写入中央库并创建快照。
6. 再逐个平台发布。
7. 结果按平台显示成功或失败。

### 9.5 编辑并同步

1. 用户打开中央 Skill 编辑工作区。
2. 修改文件并查看差异。
3. 首次保存前应用创建恢复点。
4. 保存后 Skill 进入“有未发布变更”状态。
5. 用户选择快照和目标 Agent。
6. 如果目标被外部修改，应用显示漂移冲突。
7. 用户选择导入目标变更、覆盖目标或取消。

### 9.6 从单个平台移除

1. 用户打开 Skill 的安装位置。
2. 选择某个平台并点击“移除”。
3. 应用明确提示中央主副本不会删除。
4. 移除目标映射并记录操作。

### 9.7 删除和恢复

1. 用户点击删除中央 Skill。
2. 应用展示受影响映射和删除计划。
3. 用户确认后，Skill 移入应用回收站。
4. 回收站中保留原路径和关联信息。
5. 用户点击恢复；无冲突时恢复原位。
6. 有冲突时用户选择新名称或取消。
7. 永久删除只能在回收站中二次确认后执行。

### 9.8 配置模型

1. 用户进入“模型与 API”。
2. 配置 MiniMax API Key、API 地址和采集模型。
3. 配置 OpenAI API Key 和 `gpt-5.6`。
4. 分别执行连接测试。
5. 页面显示 Provider 状态、实际模型和职责。
6. 用户可修改路由或关闭某一类 AI 功能。

## 10. AI 产品行为

### 10.1 MiniMax 负责

- 从长 `SKILL.md` 中提取用法要点
- 识别可能的标签和类别
- 抽取环境要求、命令、输入与输出提示
- 为 GPT-5.6 提供结构化候选信息

### 10.2 GPT-5.6 负责

- 生成最终一句话简介
- 将复杂用法提炼为用户可快速理解的说明
- 解释同名冲突和版本差异
- 在证据冲突时生成可读解释，但不修改确定性可信度分数

### 10.3 用户可见信息

任何 AI 内容旁显示：

- 服务商
- 实际模型 ID
- 负责的内容
- 生成时间
- 最新/过期状态
- 重新生成入口

## 11. 来源可信度的产品表达

来源卡片包含：

- 结论：“GitHub 仓库”“Codex 系统 Skill”“插件提供”或“未知”
- 状态：已确认、推断、未知
- 可信度：例如 92%
- 主要依据：例如“安装记录匹配 + Git remote 匹配”
- 其他证据
- 用户纠正入口

颜色不得作为唯一表达方式。可信度不是“安全评分”，页面必须避免混淆。

### 12.1 品牌

- 名称：Skill Studio Pro

## 14. 成功指标

第一代开源版本以本地、匿名、无需遥测的方式评估：

1. 受支持测试目录中的 Skill 发现准确率达到 99%。
2. 有可验证来源的测试样本中，来源类型判断准确率达到 95%。
3. 从首次启动到看到首批索引结果不超过 60 秒（500 个 Skill 基准集）。
4. 核心安装、编辑、同步、删除、恢复流程无不可恢复数据丢失。
5. 五个首要 Agent 的契约测试全部通过。
6. 离线状态下，除 AI 和网络导入外的核心功能可使用。
7. 自动化测试覆盖全部 P0 验收标准。

应用不默认收集匿名使用数据。项目通过公开 Issue、测试报告和用户自愿反馈评估体验指标。

## 15. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| Agent 目录规则持续变化 | 扫描遗漏 | 平台 Adapter、可配置目录、契约测试 |
| 同名 Skill 多版本 | 错误覆盖 | 冲突预览、稳定内部 ID、禁止自动合并 |
| 符号链接跨平台差异 | 映射失败 | 默认复制、能力检测、明确降级 |
| 外部目录权限不足 | 无法扫描或写入 | 只读状态、错误提示、选择新目录 |
| AI 结果不准确 | 用户误判 | 原文优先、结果归属、过期标记、可关闭 |
| API Key 泄露 | 严重安全事件 | 系统凭据存储、日志脱敏、秘密扫描 |
| 递归删除路径错误 | 数据丢失 | 计划预览、路径边界校验、回收站、二次确认 |
| 上游更新导致分叉困难 | 维护成本上升 | 保持模块边界、记录基线、定期上游同步 |
| Apple 品牌素材授权限制 | 发布风险 | 只借鉴原则，使用自有品牌资产 |

## 16. 发布与开源要求

1. 公开源代码和构建说明。
2. 保留 Apache-2.0 LICENSE、NOTICE 和上游归属。
3. README 说明 Skill Studio Pro 与上游 Skill Studio 的关系。
4. 发布包不包含用户 Skill、数据库、日志和 API Key。
5. 提供 SECURITY、CONTRIBUTING、第三方依赖清单和负责任披露方式。
6. Windows、macOS、Linux Release 附校验文件。

## 17. 产品验收原则

本 PRD 中的页面和流程不得改变 [SPEC.md](./SPEC.md) 定义的安全语义。尤其是：首次扫描只读、来源事实与推断分离、默认复制映射、删除进入回收站、AI 可选、密钥不落明文和团队功能不进入第一代范围。

## 18. Draft 0.2 当前状态与下一阶段

### 18.1 当前产品阶段

以 `d626224` 为实现基线，Skill Studio Pro 已达到 Windows 内部 Beta 候选：核心 Rust 服务、SQLite 数据模型、真实 Tauri IPC、深海蓝桌面壳、Windows NSIS 打包和自动化安全门槛已经形成。

当前仍不能称为公开跨平台 V1。原因不是底层架构缺失，而是三个高频用户流程尚未完整进入界面，以及公开发布所需的真实平台证据尚不完整。

### 18.2 V1 产品闭环冲刺

下一轮 P0 按以下顺序完成：

1. **编辑闭环**：中央 Skill 的文件树、只读/编辑模式、格式校验、未保存保护、保存、恢复点、差异和映射过期反馈。
2. **AI 生成闭环**：在详情页生成简介和用法、显示进度、取消、错误、缓存、过期和重新生成；展示实际模型与职责。
3. **平台与扫描根闭环**：平台检测、路径测试、目录选择、启停、同步模式、受管数量、影响预览和自定义扫描根管理。
4. **真实 Windows 验收**：使用隔离 Home、中央库和 Agent 目录，在真实桌面 UI 完成“扫描→纳管→编辑→AI→发布→漂移→回收→恢复”。
5. **公开 Beta 发布**：独立公开仓库、三平台 CI、三平台构建/启动证据、SBOM、第三方清单、安装文档和 Beta Release。

### 18.4 关键页面完成定义

#### 编辑工作区

- 外部实例只读，编辑前必须纳管。
- 支持 Markdown、YAML、JSON、TOML 和纯文本。
- 切换文件、路由或退出时保护未保存内容。
- 保存失败保留编辑内容，保存成功显示恢复点和过期映射数量。
- 用户可查看保存前后差异并继续发布。

#### AI 简介与用法

- 没有缓存时显示“生成简介/用法”，过期时显示“重新生成”。
- 运行中显示实际任务、Provider、模型和取消操作。
- 错误区分凭据、网络、限流、配额、模型和结构问题，并给出下一步。
- 自动补充是显式可选设置，默认关闭；离线功能不受影响。

#### 平台中心与扫描根

- 五个首要 Agent 使用真实检测和配置数据。
- 用户可以测试目录并保存，路径无效时不可假装成功。
- 修改或停用已有映射的平台前显示治理影响。
- 自定义扫描根的移除不得删除磁盘内容。

### 18.5 发布阶段

1. **Internal Beta**：上述三个产品闭环完成，Windows 真实沙箱流程通过。
2. **Public Beta (`v0.1.0-beta.1`)**：公开仓库、三平台 CI 和构建/启动证据、文档与依赖清单完成。
3. **Stable Candidate**：高 DPI/可访问性人工验收、大规模性能基准、真实 Secret Store、签名/公证策略和 Beta 缺陷回归完成。

任何阶段都不得用 Mock 页面、静态平台卡片或仅后端存在的命令作为用户流程完成证据。
